零基础教学

从电脑上什么都没有开始，一步步搭建Vue3开发环境并创建第一个项目

1

安装 Node.js 和 npm — Vue 的运行基础

▼

什么是 Node.js？为什么要装它？

Vue3 项目需要用 npm（Node Package Manager）来下载各种依赖包（比如 vue、vue-router 等）。而 npm 是跟着 Node.js 一起安装的，所以第一步就是安装 Node.js。

简单理解：Node.js 是发动机，npm 是工具箱，Vue 是你要用的零件。先装发动机，才能用工具箱，才能装零件。

安装步骤（Windows）

  1. 打开浏览器，访问 https://nodejs.org
  2. 下载 LTS（长期支持版），一般是 .msi 文件
  3. 双击安装包，一路点"下一步"，不需要改任何设置
  4. 安装完成后，打开 命令行（按 Win+R，输入 cmd，回车）
  5. 输入以下命令验证安装成功：
bash复制
node -v     # 显示版本号，如 v20.10.0
npm -v      # 显示版本号，如 10.2.3
提示：看到版本号就说明安装成功。如果提示"不是内部或外部命令"，重启电脑。

npm 常用命令速查

命令作用
npm install 包名安装一个包到当前项目
npm install 包名 -g全局安装（如脚手架工具）
npm install安装 package.json 中所有依赖
npm update 包名更新指定包
npm uninstall 包名卸载包
npm run dev启动开发服务器
npm run build构建生产版本
易错点：yarn dev / npm run dev 是启动开发服务器，不是项目构建/打包。npm run build 才是构建。

yarn 是什么？

yarn 是 Facebook 开发的另一个包管理工具，功能和 npm 一样。两者可以互相替代：

npmyarn作用
npm installyarn安装所有依赖
npm install 包名yarn add 包名安装指定包
npm run devyarn dev启动开发服务器

使用 yarn 前也需要先安装 Node.js。安装 yarn：npm install -g yarn

npm 安装同一个包时会缓存，再次安装无须重复下载。npm 可以并行安装多个包，不需要等到当前包安装完才继续。

2

安装 Vue 脚手架并创建项目

▼

什么是脚手架？

Vue 脚手架（Vite）是一个工具，帮你自动生成一个完整的 Vue 项目目录结构，不用手动一个一个建文件。

类比：脚手架就像装修时的样板房模板 — 一键生成所有房间布局，你只需要往里面填东西就行。

创建 Vue3 项目

打开命令行，进入你想创建项目的文件夹，执行：

bash复制
# 方式1：使用 npm 创建（推荐）
npm create vite@latest my-vue-app -- --template vue

# 方式2：使用 yarn 创建
yarn create vite my-vue-app --template vue

然后进入项目目录并安装依赖：

bash复制
cd my-vue-app
npm install    # 或 yarn
npm run dev    # 启动开发服务器

浏览器打开 http://localhost:5173 就能看到你的第一个 Vue 页面了！

项目目录结构

目录结构
my-vue-app/
├── node_modules/     # 下载的依赖包（自动生成）
├── public/           # 静态文件
├── src/              # ★ 你写代码的地方
│   ├── components/   # 组件文件夹
│   ├── App.vue       # ★ 根组件（页面入口）
│   ├── main.js       # ★ 入口文件（挂载Vue）
│   └── style.css     # 全局样式
├── index.html        # HTML 模板
├── package.json      # 项目配置和依赖清单
└── vite.config.js    # Vite 构建配置

.vue 文件的三段式结构

每个 .vue 文件就是一个组件（SFC，Single File Component），包含三个部分：

Vue SFC 结构复制
<template>
  <!-- HTML 结构：写页面长什么样 -->
  <div>
    <h1>{{ message }}</h1>
  </div>
</template>

<script setup>
  // JS 逻辑：写数据和功能
  import { ref } from 'vue'
  const message = ref('Hello Vue3!')
</script>

<style scoped>
  /* CSS 样式 */
  h1 { color: #42b883; }
</style>
关键：<template> 写 HTML，<script setup> 写 JS，<style> 写 CSS。这三段是每个 .vue 文件的标准结构。
注意：Vue3 中 <template> 可以有多个根节点（Vue2 只能有一个）。考试说"只能有一个根节点"是错误的。

Vue 核心概念 — MVVM 模式

▼

什么是 MVVM？

MVVM = Model - View - ViewModel

Model
数据层
↔
ViewModel
Vue框架（桥梁）
↔
View
页面（DOM）
  • Model（模型）：负责业务数据的处理 — 就是你的数据（变量、数组等）
  • View（视图）：负责视图的处理 — 就是用户看到的 HTML 页面
  • ViewModel（视图模型）：负责监听 Model 和 View 的改变 — 就是 Vue 框架本身
易错点：Model 和 View 不能直接通信！它们必须通过 ViewModel 中转。考试说"Model和View可以直接通信"是错误的。

Vue 的四大特性

特性说明
轻量级Vue 相比 Angular 和 React 更小更快
数据驱动视图数据变了，页面自动更新，不用手动操作 DOM
双向数据绑定输入框输入 → 数据自动变；数据变 → 页面自动更新
插件化开发Vue Router（路由）、Pinia（状态管理）等都是插件
考试重点：Vue 自定义指令以 v- 开头（如 v-if、v-for），不是以 "on-" 开头。说"以on-开头"是错误的。

第一个 Vue 程序 — 从零写代码

▼

修改 App.vue 写你的第一个程序

打开 src/App.vue，替换为以下代码：

App.vue复制
<template>
  <div>
    <h1>{{ title }}</h1>
    <input v-model="title" placeholder="输入文字" />
    <p>你输入了：{{ title }}</p>
  </div>
</template>

<script setup>
import { ref } from 'vue'
const title = ref('我的第一个Vue程序')
</script>

<style>
h1 { color: #42b883; text-align: center; }
</style>

逐行解释：

  1. {{ title }} — 插值表达式（Mustache 语法），把 JS 变量显示到 HTML 中
  2. v-model="title" — 双向绑定，输入框内容和 title 变量同步
  3. ref('...') — 创建响应式数据（数据变了页面自动更新）
  4. import { ref } from 'vue' — 从 vue 中引入 ref 函数
验证：保存后浏览器自动刷新（热更新），在输入框打字，下面的文字会同步变化 — 这就是双向数据绑定！

ref vs reactive — 两种创建响应式数据的方式

ref()reactive()
用途基本类型（字符串、数字、布尔）对象/数组
访问JS中需要 .value，模板中不用直接访问属性
示例const count = ref(0)const state = reactive({count:0})
考试重点：
  • ref() — 将基本类型数据转为响应式
  • reactive() — 将对象/数组的所有属性转为响应式
  • toRef() — 将响应式对象的单个属性转换为 ref
  • toRefs() — 将响应式对象的所有属性转换为 ref
易错点：toRef() 是把响应式对象的属性转成 ref，不是把普通数据转成响应式。说"toRef()用于将普通数据转换成响应式数据"是错误的。

初识 Vue.js · MVVM · npm · 包管理

▼

Vue.js 是什么？

Vue 是一个用于构建用户界面的 渐进式 JavaScript 框架。核心库只关注视图层，容易上手。与 Angular、React 并称前端三大框架。

  • Vue 相比 Angular 和 React 而言，是一个轻量级的前端库
  • Vue 支持 Pinia 状态管理插件
  • Vue 支持 双向数据绑定
  • Vue 自定义指令以 v- 开头（如 v-if、v-for、v-bind）

npm 包管理工具

  • npm 是一个包管理工具，用来解决 Node.js 代码部署问题
  • npm 基于 Node.js 运行环境（不是 Vue、不是 Babel）
  • 使用 npm 安装同一个包时会缓存，再次安装无须重复下载
  • npm 可以并行安装多个包（不需要等到当前包安装完才继续）
  • npm install 包名 -g 表示全局安装，不是安装到当前项目
  • 使用 npm 前必须先安装 Node.js
易错点："npm安装包时必须等到当前包安装完成后才会继续后面的安装"是错误的 — npm 支持并行安装。

MVVM 模式详解

组成部分职责
Model主要负责业务数据的处理
View负责视图的处理（页面展示）
ViewModel负责监听 Model 或 View 的改变（Vue 框架）
易错点：Model 和 View 不能直接通信、互相监控。它们必须通过 ViewModel 中转。

Vue 特性总结

特性说明
轻量级体积小、速度快
数据驱动视图数据变化自动更新视图
双向数据绑定v-model 实现数据和视图同步
插件化开发Vue Router、Pinia 等以插件形式接入

Vue 开发基础 · 指令 · 响应式数据 · 事件绑定

▼

常用指令一览表

指令作用示例
v-text渲染 DOM 元素的文本内容v-text="msg"
v-html渲染 HTML 内容v-html="rawHtml"
v-bind（:）绑定属性:href="url"
v-on（@）绑定事件@click="fn"
v-model双向数据绑定v-model="msg"
v-if / v-else条件渲染（删除/创建DOM）v-if="show"
v-show条件显示（display切换）v-show="show"
v-for列表渲染v-for="item in list"
考试重点：渲染 DOM 元素文本内容的指令是 v-text（不是 v-bind、v-on、v-for）。

v-bind 指令 — 属性绑定

v-bind 用于绑定 HTML 属性，简写为冒号 :。可以绑定 class、style、src、href 等属性。

Vue复制
<!-- 完整写法 -->
<a v-bind:href="url">链接</a>
<div v-bind:class="{ active: isActive }"></div>

<!-- 简写 -->
<a :href="url">链接</a>
<div :class="{ active: isActive }"></div>

Vue 中绑定样式类可以通过 v-bind 指令操作 class 属性或 style 属性来实现。

v-on 指令 — 事件绑定

v-on 用于绑定事件监听，简写为 @。

Vue复制
<!-- 完整写法 -->
<button v-on:click="handleClick">点击</button>

<!-- 简写 -->
<button @click="handleClick">点击</button>

<!-- $event 获取事件对象 -->
<button @click="handleClick($event)">点击</button>
$event 是 Vue 提供的内置变量，使用它可以获取事件对象。

事件修饰符

修饰符作用
.prevent阻止默认行为（如 a 标签跳转）
.stop阻止事件冒泡
.capture改变事件执行顺序，从冒泡改为捕获
.self只有 DOM 元素本身才会触发事件
.once事件只触发一次
易错点：.stop 是阻止冒泡，不是阻止默认事件行为。阻止默认行为的是 .prevent。

v-model 修饰符

修饰符作用
.lazychange 事件时同步（非 input 事件）
.number自动将输入转为数字类型
.trim自动过滤首尾空白字符
考试重点：v-model.trim 可以自动过滤用户输入的首尾空白字符。

v-for 指令 — 列表渲染

v-for 可以基于数组、对象、数字或字符串循环渲染列表。

Vue复制
<!-- 数组遍历 -->
<li v-for="(item, index) in list" :key="item.id">
  {{ item.name }}
</li>

<!-- 对象遍历 -->
<div v-for="(value, key) in obj" :key="key">
  {{ key }}: {{ value }}
</div>
易错点：使用 v-for 时必须指定 key 的值，且 key 的值必须具有唯一性。说"key的值不具有唯一性"是错误的。
v-for 循环次数：数组 → 按数组元素个数循环；对象 → 按对象属性个数循环。

响应式数据函数总结

函数作用说明
ref()将基本类型转为响应式访问时需要 .value
reactive()将对象所有属性转为响应式直接访问属性
toRef()将响应式对象的单个属性转为 ref保持引用关系
toRefs()将响应式对象所有属性转为 ref解构时使用
易错点：ref() 用于将基本类型数据转为响应式，不是"将响应式对象中的单个属性转换为响应式数据" — 那是 toRef() 的功能。

单文件组件（SFC）

  • 模板（template）：用于搭建当前组件的 DOM 结构
  • 样式（style）：通过 CSS 代码为当前组件设置样式
  • 逻辑（script）：通过 JavaScript 代码处理组件的数据与业务
易错点：Vue 3 中 <template> 标签中的 DOM 结构可以有多个根节点（Vue2 只能有一个）。说"只能有一个根节点"是错误的。

组件基础(上) · Props · 生命周期 · 依赖注入

▼

组合式 API 与 setup

Vue3 推荐使用组合式 API（Composition API），在 <script setup> 中直接定义数据和方法。

组合式 API：数据和方法可以直接在 setup() 函数中定义。<script setup> 是 setup() 函数的语法糖。

生命周期函数

函数触发时机
onBeforeMount()组件挂载之前被调用
onMounted()组件挂载完成后被调用
onBeforeUpdate()组件更新前被调用
onUpdated()组件更新完成后被调用
onBeforeUnmount()组件卸载前被调用
onUnmounted()组件实例被销毁后调用
易错点：onUpdated() 是在组件更新完成后被调用，不是更新前。说"onUpdated()在组件更新前被调用"是错误的。

Props — 父传子数据

Props 是父组件向子组件传递数据的方式。在子组件中用 defineProps() 声明。

子组件复制
<script setup>
const props = defineProps({
  title: {
    type: String,
    default: '默认标题',
    required: true
  }
})
</script>
  • 可以通过 type 属性对传递过来的 props 数据进行基础类型检查
  • 通过 default 属性设置默认值
  • 通过 required 属性设置必填项
  • 所有的 props 都遵循单向数据流原则（子组件不能直接修改父组件传来的 props）
易错点：对象形式的 props 可以使用多种验证方案（type、default、required、validator 等）。说"不能使用多种验证方案"是错误的。
易错点：在父组件中使用 v-bind 可以为子组件动态绑定 props（不是"静态"绑定）。说"静态绑定"是错误的。

依赖注入 — 跨级组件通信

跨级组件（祖孙组件）之间的数据传递可以通过依赖注入来实现。

祖辈组件复制
import { provide } from 'vue'
provide('key', value)  // 提供数据
后代组件复制
import { inject } from 'vue'
const value = inject('key')  // 注入数据
  • provide() 函数提供一个值，可以被后代组件所注入
  • inject() 函数用于注入上层组件提供的数据
易错点：provide() 接收 2 个参数：第1个是注入名（key），第2个是要注入的值（value）。说"第1个参数是要注入的值"是错误的。

自定义事件 — 子传父数据

子组件通过自定义事件向父组件传递数据。在子组件中用 defineEmits() 声明。

子组件复制
<script setup>
const emit = defineEmits(['demo'])
const sendData = () => {
  emit('demo', data)  // 触发事件并传数据
}
</script>
父组件复制
<Child @demo="handleDemo" />
考试重点：在 setup 语法糖中声明自定义事件的方式是 defineEmits()。监听自定义事件的指令是 v-on（即 @）。
4

组件基础(下) · 插槽 · 动态组件 · 自定义指令 · KeepAlive

▼

动态组件

使用 <component :is="组件名"> 实现组件之间的按需切换。

Vue复制
<component :is="currentComponent"></component>

使用动态组件切换时，隐藏的组件会被销毁，展示出来的组件会被重新创建。

KeepAlive — 组件缓存

用 <KeepAlive> 标签包裹组件，可以使组件创建一次后不会被销毁。

Vue复制
<KeepAlive include="CompA,CompB" :max="10">
  <component :is="currentComp" />
</KeepAlive>
  • include 属性 — 只有匹配的组件名才会被缓存
  • exclude 属性 — 匹配的组件名不会被缓存
  • max 属性 — 最多缓存的组件实例个数
易错点：若只想要对应组件名的组件被缓存，应使用 include 属性，不是 exclude 属性。说"通过exclude实现"是错误的。
组件缓存时触发的生命周期函数：onActivated()（激活时）和 onDeactivated()（停用时）。

插槽（Slot）

插槽是组件封装期间为组件使用者预留的占位符。

  • 默认插槽：直接写 <slot> 标签
  • 具名插槽：添加 name 属性的 <slot> 标签，如 <slot name="header">
  • 作用域插槽：带有数据的插槽，子组件提供数据给父组件
易错点：如果一个组件没有预留任何插槽，则组件使用者提供的插槽内容不会起作用（会被丢弃）。说"同样会起作用"是错误的。
易错点：如果组件的使用者为插槽提供了内容，则使用者提供的内容生效，默认内容不生效。说"默认内容生效"是错误的。
易错点：作用域插槽是子组件提供数据给插槽（不是父组件提供）。说"父组件提供一部分数据给插槽"是错误的。

自定义指令

  • 全局自定义指令：通过 app.directive() 声明，可以在全局使用
  • 私有自定义指令：在组件的 directives 选项中声明，只能在该组件中使用
  • 可以为自定义指令绑定参数（如 v-focus:foo）
易错点：自定义指令可以绑定参数。说"不能为自定义指令绑定参数"是错误的。

自定义指令的生命周期函数

函数触发时机
mounted()绑定元素的父组件及自身的所有子节点都挂载完成后调用
beforeUpdate()绑定元素的父组件更新前调用
updated()绑定元素的父组件及子节点更新完成后调用
易错点：beforeUpdate() 是在绑定的父组件更新前调用（不是卸载前）。说"卸载前调用"是错误的。
参数说明：el — 当前指令所绑定到的元素；binding.value — 传递给指令的值。

路由 · Vue Router · 导航守卫 · 编程式导航

▼

Vue Router 基础

Vue Router 是 Vue.js 的官方路由管理器，用于实现 SPA（单页面应用）的路由功能。

安装命令：npm install vue-router@4 或 yarn add vue-router@4

易错点：正确的安装命令是 npm install vue-router@4 或 yarn add vue-router@4。选项中 "npm I vue-router@4" 的大小写可能被当作错误写法。

前端路由 vs 后端路由

前端路由后端路由
发生位置浏览器端服务器端
示例Vue 中的路由Express 框架中的路由
易错点：Vue 中的路由属于前端路由，不是后端路由。

路由核心概念

  • <router-link>：声明式导航链接（渲染为 a 标签）
  • <router-view>：路由视图出口（渲染匹配的组件）← 不是 router-link！
  • routes：路由匹配规则数组
  • redirect：路由重定向属性
  • router-link-active：默认的激活路由链接高亮类名
易错点：路由视图使用 <router-view> 标签定义，不是 <router-link>。说"路由视图使用router-link定义"是错误的。
易错点：页面有两种导航方式：声明式导航和编程式导航（不是"嵌套式"导航）。

导航守卫

Vue Router 提供导航守卫来控制路由的访问权限。

全局前置守卫复制
const router = createRouter({...})
router.beforeEach((to, from, next) => {
  // to: 目标路由对象
  // from: 当前导航正要离开的路由对象
  // next: 放行函数
  next()
})
易错点：beforeEach() 接收 to、from、next 三个形参。若省略 next 参数，Vue Router 4 中允许访问路由（不是"不允许访问任何路由"）。

编程式导航

编程式导航复制
router.push('/path')     # 跳转到指定路由
router.go(1)               # 向前移动1条记录
router.go(-1)              # 向后移动1条记录
易错点：router.go(-1) 表示向后移动1条记录（不是2条）。说"向后移动两条记录"是错误的。router.go() 类似 window.history.go()。

命名路由

通过 name 属性给路由规则命名，name 属性值不能重复。

命名路由复制
// 路由定义
{ path: '/user', name: 'user', component: User }

// 声明式导航使用
<router-link :to="{ name: 'user', params: { id: 1 } }"></router-link>
易错点：命名路由的 name 属性值不能重复。说"name属性值可以重复"是错误的。

网络请求和状态管理 · Axios · Vuex · Pinia

▼

Axios

Axios 是基于 Promise 的 HTTP 库，可以作用于 Node.js 和浏览器中。

  • 在浏览器中使用 XMLHttpRequest 接口发送请求
  • 在 Node 环境中使用 Node.js 的 http 模块发送请求
  • 支持拦截请求和响应
  • 可以转换请求数据和响应数据
  • 支持 Promise API
  • 支持大多数的 API
  • 支持 post 请求
易错点：Axios 支持 post 请求。说"不支持post请求"是错误的。
易错点：Axios 的安全性：客户端支持防御 XSRF（不是 CSRF）。说"支持防御CSRF"是错误的。

Vuex 状态管理

Vuex 是专为 Vue.js 应用程序开发的状态管理库。

核心概念作用
state管理数据，数据是响应式的，数据改变驱动视图更新
mutations更新数据，state 中的数据只能通过 mutations 改变
actions定义事件处理方法，把数据提交给 mutations，不能直接修改 state
modules定义模块对象
易错点：actions 不能直接修改 state，必须通过 mutations 来修改。说"actions可以直接修改state"是错误的。

Pinia 状态管理

Pinia 是 Vue 的新一代状态管理库，支持 Vue 3 和组合式 API。

  • Pinia 提供了完整的 TypeScript 支持
  • Pinia 没有 modules 概念（每个 store 独立存在），代码更加简洁
  • Pinia 支持 Devtools 调试工具
  • Pinia 支持服务端渲染
易错点：Pinia 不使用 modules 划分模块。说"Pinia使用modules划分模块"是错误的。
易错点：Pinia 支持 Vue 2（不仅支持 Vue 3）。说"不支持Vue2"是错误的。
易错点：Pinia 中通过 paths 实现部分数据持久化存储。

题3：创建组件，完成音乐列表增删改查功能

▼

题目要求

创建组件，实现音乐播放列表的增删改查功能。包含：添加歌曲、播放（增加播放次数）、重置、计算总播放时长。

完整代码

MusicList.vue复制
<template>
  <div>
    <h2>音乐播放列表</h2>
    <div>
      <input type="text" v-model="state.song.name" placeholder="请输入歌曲名称">
      <input type="number" v-model="state.song.duration" placeholder="请输入时长(秒)">
      <button @click="submitSong">确认添加歌曲</button>
    </div>
    <div>
      <table>
        <thead>
          <tr>
            <th>歌曲</th>
            <th>时长(秒)</th>
            <th>播放次数</th>
            <th>播放操作</th>
          </tr>
        </thead>
        <tbody>
          <tr v-for="(item, index) in state.songs" :key="index">
            <td>{{ item.name }}</td>
            <td>{{ item.duration }}秒</td>
            <td>{{ item.playCount }}</td>
            <td>
              <button @click="play(index)">播放</button>
              <button @click="stop(index)">重置</button>
            </td>
          </tr>
          <tr>
            <td>总播放时长：</td>
            <td colspan="3">{{ totalDuration }}秒</td>
          </tr>
        </tbody>
      </table>
    </div>
  </div>
</template>

<script setup>
import { reactive, computed } from 'vue'

const state = reactive({
  song: {
    name: '',
    duration: 0,
    playCount: 0
  },
  songs: [
    { name: '稻香', duration: 223, playCount: 5 },
    { name: '夜曲', duration: 198, playCount: 3 },
    { name: '青花瓷', duration: 235, playCount: 4 }
  ]
})

const submitSong = () => {
  if (state.song.name == '' || state.song.duration <= 0) {
    alert('请输入歌曲名称且时长需大于0秒')
  } else {
    state.songs.push({ ...state.song, playCount: 0 })
    state.song = { name: '', duration: 0, playCount: 0 }
  }
}

const play = (index) => {
  state.songs[index].playCount++
}

const stop = (index) => {
  state.songs[index].playCount = 0
}

const totalDuration = computed(() => {
  return state.songs.reduce((total, item) => {
    return total + (item.duration * item.playCount)
  }, 0)
})
</script>

关键知识点讲解

  1. v-model 双向绑定：v-model="state.song.name" 输入框和数据同步
  2. 添加功能：state.songs.push({...}) 向数组添加新歌曲
  3. 展开运算符：{...state.song, playCount: 0} 复制 song 对象并重置 playCount
  4. 播放功能：通过 index 找到对应歌曲，playCount++ 自增
  5. 重置功能：通过 index 找到对应歌曲，playCount = 0
  6. computed 计算属性：totalDuration 自动计算总播放时长 = 每首歌时长 × 播放次数之和
  7. reduce 方法：遍历数组累加，初始值 0
考点：reactive、computed 计算属性、v-model 双向绑定、数组 push/splice、@click 事件绑定、展开运算符
