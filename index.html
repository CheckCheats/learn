ref vs reactive 必考

函数用途JS中访问
ref()基本类型（字符串/数字/布尔）.value
reactive()对象/数组直接访问
toRef()提取reactive单个属性.value
toRefs()reactive所有属性→ref.value
核心区别：ref在JS中用 .value，模板中自动解包不需要。reactive不能用于基本类型！

组件基础（上）· 生命周期 · Props · 事件

▼

生命周期 必考

setup() ← 最先执行 ↓ onBeforeMount() ← 挂载前 ↓ onMounted() ⭐ ← 挂载完成（操作DOM / 请求数据） ↓ onBeforeUpdate() ← 更新前 ↓ onUpdated() ← 更新后 ↓ onBeforeUnmount() ← 卸载前 ↓ onUnmounted() ← 卸载后（清理定时器） ── KeepAlive 缓存 ── onActivated() ← 缓存组件激活 onDeactivated() ← 缓存组件失活
⭐ onMounted 最重要！DOM已渲染完毕，常用于：请求数据、操作DOM、初始化第三方库。

Props 父→子 必考

vue - 子组件
<script setup>
const props = defineProps({
  title: { type: String, required: true },
  count: { type: Number, default: 0 }
})
</script>
vue - 父组件
<Child :title="pageTitle" :count="num" />
Props单向数据流！子组件不能直接修改props。

自定义事件 子→父 必考

vue - 子组件 emit
const emit = defineEmits(['change'])
emit('change', 100)  // 触发并传值
vue - 父组件监听
<Child @change="handleChange" />
父 ── props ──→ 子 ── emit ──→ 父

provide / inject 跨级传值

vue - 祖先
import { ref, provide } from 'vue'
const money = ref(1000)
provide('money', money)  // 提供
vue - 后代
import { inject } from 'vue'
const money = inject('money')  // 注入
易错：provide第1个参数是注入名(key)，第2个是值！不是反过来。

scoped 样式隔离

vue
<style scoped>
  h5 { color: red; }  /* 只对当前组件生效 */
  :deep(.child) { color: blue; }  /* 穿透到子组件 */
</style>

三种类型：默认插槽 → 具名插槽 → 作用域插槽

<!-- 父组件接收 -->
<template #default="{ items }">
  <li v-for="i in items">{{ i }}</li>
</template>
易错：如果子组件没有 <slot>，父组件的内容会被丢弃！

const routes = [
  { path: '/', redirect: '/home' },        // 重定向

Vuex 核心概念

Actions ──commit──→ Mutations ──→ State ──→ Getters (异步) (同步修改) (状态) (计算属性)
js - store
import { createStore } from 'vuex'
export default createStore({
  state:     { count: 0 },
  mutations: { inc(s) { s.count++ } },
  actions:   { asyncInc({commit}) { setTimeout(() => commit('inc'), 1000) } },
  getters:   { double(s) { return s.count * 2 } }
})
Vuex规则：必须通过 commit('mutation名') 修改state！直接改state是错的。

Pinia（推荐替代Vuex）必考

VuexPinia
Mutations有没有（直接改state）
模块化嵌套modules独立store文件
体积较大更小
js - store/counter.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useCounter = defineStore('counter', () => {
  const count = ref(0)
  const double = computed(() => count.value * 2)
  const inc = () => { count.value++ }
  return { count, double, inc }
})
vue - 组件使用
const counter = useCounter()
counter.inc()        // 调用action
counter.count          // 访问state
counter.double         // 访问getter

软件测试基础 · 需求分析 · 计划 · 模型 · 质量

▼

什么是测试需求 必考

测试需求是指从用户需求中提取出来的、需要通过测试来验证的功能点和非功能点。

为什么要做需求分析？

  • 确保测试覆盖所有需求点，避免遗漏
  • 为编写测试用例提供依据
  • 帮助评估测试工作量和测试范围
合法来源：需求规格说明书、合同/协议、行业标准/规范。❌ 不能作为来源：个人经验、竞争对手说明书、口头承诺。

需求分析完整流程

需求获取 → 需求分析 → 需求评审 → 需求验证 → 需求跟踪

需求跟踪矩阵 高频

需求跟踪矩阵（RTM）是将需求与测试用例进行映射的表格，确保每条需求都有对应的测试用例覆盖。

需求ID需求描述测试用例测试状态
REQ-001用户登录功能TC-001, TC-002, TC-003✅ 通过
REQ-002密码修改功能TC-004, TC-005⏳ 待测
作用：追踪需求覆盖情况、发现遗漏需求、辅助回归测试。

测试计划 必考

测试计划是对测试活动的全面规划，包括：测试范围、测试策略、资源安排、进度计划、风险评估、出入口准则。

编制原则：尽早制定、逐步细化、可执行可度量。
❌ 错误做法：跳过测试计划直接写用例、测试范围不明确。

软件测试模型 必考

模型核心思想适用场景
V模型测试与开发阶段一一对应（需求→验收、概要→集成、详细→单元）需求明确、小项目
W模型双V模型，开发V+测试V并行，强调测试贯穿整个生命周期中大型项目、重视质量
H模型测试是独立流程，可以随时插入，与其他流程并行敏捷开发、迭代频繁
X模型探索性测试 + 编码/测试交织进行需求不明确、原型开发
易错：V模型的"V"左边是开发，右边是测试，一一对应！W模型比V模型多了"测试与开发并行"的思想。

软件质量模型 GB/T25000.10-2016 必考

八大质量特性：

编号特性含义

常见HTTP方法 必考

方法用途特点
GET获取资源参数在URL中，幂等
POST创建资源参数在body中，非幂等
PUT全量更新幂等
PATCH部分更新非幂等
DELETE删除资源幂等
幂等：多次请求结果一样（GET/PUT/DELETE）。非幂等：每次请求可能产生不同结果（POST/PATCH）。

HTTP状态码 必考

范围含义常见

3xx重定向301 永久重定向、302 临时重定向
4xx客户端错误400 Bad Request、401 未授权、403 禁止、404 未找到
5xx服务器错误500 服务器内部错误、502 网关错误、503 服务不可用

单元测试 · unittest框架

▼

测试级别对比 必考

级别测试对象执行者
单元测试最小模块（函数/方法）开发人员
集成测试模块间的接口和交互开发/测试
系统测试整个系统功能和非功能测试团队
验收测试用户需求验证用户/客户
单元测试特点：粒度最小、执行最快、成本最低、由开发人员自己完成。

unittest核心结构 必考

Python
import unittest

class TestMath(unittest.TestCase):
    def setUp(self):
        # 每个测试方法执行前运行（初始化）
        self.data = [1, 2, 3]

    def tearDown(self):
        # 每个测试方法执行后运行（清理）
        self.data = None

    def test_sum(self):
        self.assertEqual(sum(self.data), 6)

    def test_len(self):
        self.assertEqual(len(self.data), 3)

# 执行测试
if __name__ == '__main__':
    unittest.main()

💡 解题思路

  • 用 while True 无限循环，直到猜对用 break 跳出

→
用户输入
→
判断大小
→
提示/跳出
→
输出次数

💻 完整代码

python
import random

target = random.randint(1, 100)

学生类 · 面向对象 · class

▼

📝 题目要求 必考

定义一个学生类（student），包含姓名、学号、成绩属性。编写方法判断成绩等级（优秀/良好/中等/合格/不及格），并输出学生信息。

💡 解题思路

  • __init__ 构造函数初始化属性
  • level() 方法用 if-elif-else 判断成绩区间
  • info() 方法用 f-string 格式化输出
  • if __name__ == '__main__' 写测试代码
成绩范围等级
90 ~ 100优秀
80 ~ 89良好
70 ~ 79中等
60 ~ 69合格
< 60不及格

💻 完整代码

python
class student:
    def __init__(self, name, num, score):
        self.name = name
        self.num = num
        self.score = score

    def level(self):
        if self.score < 0 or self.score > 100:
            return '成绩不合法'
        elif self.score >= 90:
            return '优秀'
        elif self.score >= 80:
            return '良好'
        elif self.score >= 70:
            return '中等'
        elif self.score >= 60:
            return '合格'
        else:
            return '不及格'

    def info(self):
        l = self.level()
        print(f'我的姓名是{self.name}, 我的学号是{self.num}, 我的成绩等级是{l}')

if __name__ == '__main__':
    s1 = student('张三', '123456', 100)
    s1.info()
知识点：self 代表实例自身，__init__ 是构造方法，__name__ == '__main__' 确保直接运行才执行。
易错：每个方法的第一个参数必须是 self，调用时不需要传！

核心知识点速记 · 100题精华提炼 · 考前必背

▼

📱 基础概念与配置

  • slider = 滑动选择器（左右滑动选数值），不是 progress/input
  • app.json pages 数组注册所有页面，首项 = 默认首页
  • rpx：750rpx = 整屏宽度，自动适配多端；大写 Rpx 无效
  • app.js 中 this 指向 App 实例，小程序没有 window 对象
  • 张小龙主导推出微信小程序
  • 微信开发者工具是官方专属开发工具
  • 页面 .json 配置优先级 > 全局 app.json（就近原则）
  • 分包加载：拆分主包资源，减少主包体积，优化首屏加载
  • 小程序免安装、依赖微信运行、支持多端适配
  • networkTimeout 统一配置网络请求/上传/下载超时时间
  • app.json = 全局配置文件（pages + window + tabBar + networkTimeout）

🧩 内置组件

  • checkbox-group 复选框分组；radio-group 单选框分组
  • <block> 逻辑包装标签，不渲染为真实 DOM 节点
  • <template> 定义可复用页面模板
  • <include> 仅引入外部模板片段，无逻辑处理能力
  • <swiper> 轮播图组件，默认关闭自动播放，需手动设 autoplay
  • <progress> = 进度条组件；<slider> = 滑动选择器
  • <image> = 图片展示组件；<view> 最基础视图容器 ≈ div
  • <slot> 组件插槽，接收外部传入的自定义内容
  • button plain=true → 镂空无边框样式
  • hover-class 设置元素按下时的样式类名
  • input type=number → 纯数字输入模式
  • <icon> 仅支持静态图标（成功/警告/加载），不能播放视频

👆 事件系统

  • tap = 单击触摸事件；touchstart/touchmove/touchend = 触摸系列
  • 事件对象包含 type / target / currentTarget / dataset，无 resource 属性
  • 事件冒泡：子元素 → 父元素；catch 绑定可阻止冒泡，bind 不阻止
  • target = 最初触发的源头元素；currentTarget = 当前绑定事件的元素
  • dataset 存放 data-* 自定义数据

🧭 路由导航

  • reLaunch 关闭所有页面，打开指定页；redirectTo 关闭当前页跳转，无法返回
  • 小程序官方路由 API：navigateTo / redirectTo / switchTab / reLaunch / navigateBack，无 forward
  • 页面栈最大 10 层，超过后 navigateTo 失效
  • setNavigationBarTitle 动态修改导航栏标题
  • setNavigationBarColor 动态修改导航栏背景色
  • hideNavigationBarLoading 关闭导航栏加载动画
  • navigateTo 不能跳转 tabBar 页面，switchTab 仅支持 tabBar 页

♻️ 生命周期

  • onLaunch 属于 App 全局生命周期，不属于 Page（仅首次打开执行一次）
  • onLoad 页面加载时触发，可接收页面跳转参数
  • App.onShow 后台切前台时触发；App.onHide 前台切后台时触发
  • onUnload 页面销毁 / 从页面栈移除时触发
  • onReady 页面初次渲染完成，DOM 就绪时触发
  • 执行顺序：onLoad → onShow → onReady → onHide → onUnload

💾 数据缓存

  • clearStorage 清空本地全部缓存数据；removeStorage 删除指定 key
  • Sync 结尾 = 同步接口（setStorageSync / getStorageSync 等）
  • 同步 API 需要 try...catch 捕获异常；异步 API 通过 fail 回调捕获
  • getStorageSync 同步读取指定 key 的缓存数据

🎵 多媒体与系统 API

  • wx.getLocation 获取当前地理位置；wx.openLocation 打开地图查看位置
  • play() 播放音频；pause() 暂停；stop() 停止；destroy() 销毁实例
  • previewImage 必须传入 urls 数组（所有预览图片链接）
  • openLocation 可调用微信地图打开位置，不是仅查看文本地址
  • sizeType 选择图片类型（原图/压缩图）；sourceType 设置来源（相册/相机）
  • 动画方法：scale() 缩放 / rotate() 旋转 / translate() 平移 / skew() 倾斜
  • wx.createAnimation 创建动画实例，支持链式调用
  • getSystemInfo 返回 brand(品牌) / model(型号) / SDKVersion(基础库) / wxVersion(微信版本)，无 systemInfo 字段
  • openDocument 支持 PDF/Word/PPT，不支持图片格式(PNG)
  • chooseImage 默认 count=1（最多选1张），支持相册+相机拍照
  • InnerAudioContext.duration = 音频总时长；currentTime = 当前播放进度
  • onPlay 监听开始播放；onStop 手动停止触发；onEnded 自然播放完毕触发
  • BackgroundAudioManager 背景音乐管理器，src 支持本地/网络地址，后台播放需 app.json 配置权限

📐 Flex 布局与 scroll-view

  • flex-direction 控制主轴排列方向（row 横向 / column 纵向）
  • justify-content 控制主轴对齐方式
  • align-items 控制侧轴（交叉轴）对齐方式
  • flex-wrap: wrap 元素空间不足时自动换行
  • scroll-y=true 开启纵向滚动；scroll-x=true 开启横向滚动
  • scroll-into-view 绑定的 id 可以数字开头，无此限制

🧩 自定义组件 · tabBar · 分享 · 杂项

  • 自定义组件支持嵌套第三方组件、内置组件，无禁止嵌套规则
  • wx:key="*this" 以数组项本身作为唯一标识；wx:key="id" 以 id 字段标识
  • wx:for 默认变量 item（数组项），index（下标）
  • 组件 json 必须配置 "component": true 声明为组件类型
  • 自定义组件默认样式隔离：外部不影响内部，内部不污染外部
  • properties 接收父组件传入的数据（不是 data）
  • tabBar 最少 2 项、最多 5 项；图标推荐 PNG 透明格式
  • tabBar 页面必须先在 app.json pages 中注册
  • onShareAppMessage 必须返回 {title, desc, path} 对象
  • 分享 desc = 描述文案；path = 点击分享卡片后跳转的页面路径
  • onReachBottom 上滑加载事件，通过 onReachBottomDistance 设置触发距离
  • wx.checkSession 校验登录态是否有效
  • button open-type="getUserInfo" 获取用户信息授权；open-type="call" 拨打电话
  • value 定义单选/复选框的选项值；checked 设置默认选中状态
  • pullDownRefresh 配置全局下拉刷新窗口背景色
  • 导航加载 API 不配对使用不会导致崩溃，仅一直显示加载动画
